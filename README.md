# EIF420O - Artificial Intelligence

## Supervised Learning Case Studies #2 and #3

**Universidad Nacional de Costa Rica**
**First Academic Term 2026**

## Overview

This repository supports the work required for Case Studies #2 and #3 in **EIF420O - Artificial Intelligence**. The case studies focus on supervised learning through experimentation with the **Potability** and **Diabetes** datasets provided by the instructor.

## Purpose

The general purpose is to develop a case study that applies prior knowledge of supervised methods through a series of experiments using the assigned datasets.

## Learning Objectives

- Research the concepts and theoretical foundations of the supervised algorithms studied in class.
- Define and evaluate multiple models using combinations of:
  - Data preprocessing or cleaning techniques
  - Feature selection techniques
  - Learning algorithms from the required groups

## Topics Covered

### Supervised Learning Algorithms

- K-Nearest Neighbors (KNN)
- Decision Trees (DT)
- Random Forest (RF)
- XGBoost
- AdaBoost

### Supporting Activities

- Data preprocessing and cleaning
- Feature selection
- Model configuration and variation analysis
- Model selection and performance comparison
- Benchmarking across algorithm families or categories

## Methodology

All required work must be implemented using the Python package developed during the course, specifically within its **Supervisados** class.

### Classification - Lab 02

For each required classification algorithm, perform an analysis using both the standard configuration and configuration variations. Select the best model and analyze it.

Required algorithms:

1. K-Nearest Neighbors
2. Decision Trees
3. Random Forest
4. XGBoost
5. AdaBoost

### Regression - Lab 03

The regression study must:

1. Identify a suitable dataset and describe its response variable and whether its predictor variables are continuous or discrete.
2. Apply benchmarking between algorithm families or categories.
3. Generate and compare models from the same algorithm family or category.
4. Generate and compare models from different algorithm families or categories.
